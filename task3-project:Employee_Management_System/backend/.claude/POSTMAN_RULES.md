# EMS Postman Rules

## Mandatory Rules

- Never delete existing requests.
- Always add new requests.
- Every new/changed endpoint must be reflected in Postman.

## Required Content Per Endpoint

- request
- headers
- body
- response

## Quality Expectations

- Collection must remain testable by the team.
- Authenticated endpoints must include Authorization header setup.
- Request bodies and expected responses must reflect current API contracts.

## Definition of Done Enforcement

- A feature is incomplete without Postman updates.
- Postman maintenance is mandatory, not optional.
